Introduction

What Is FakeNewsNet?

↗ Kaggle Dataset (mdepak) ↗ GitHub Repository

FakeNewsNet — developed at Arizona State University by Kai Shu, Deepak Mahudeswaran, Suhang Wang, Dongwon Lee, and Huan Liu — is fundamentally different from its contemporaries. While most fake news datasets stop at article text and a binary label, FakeNewsNet layers in news content, social context, visual features, and spatiotemporal information into a single unified repository. This multi-modal architecture is what sets it apart as a serious research platform rather than a simple classification benchmark.

The dataset aggregates ground-truth labels from two prestigious fact-checking organizations: PolitiFact, where journalist experts evaluate political claims, and GossipCop, which applies numerical credibility scores to entertainment news. These two sources cover radically different domains — political journalism and celebrity gossip — providing a rare cross-domain evaluation framework in a single dataset.

Why FakeNewsNet Goes Further Than ISOT/FND

Unlike ISOT's Fake & Real News Dataset, FakeNewsNet does not rely on source-level labels (Reuters = real, flagged site = fake). Instead, each article carries an individually expert-verified label. The addition of Twitter propagation graphs, user bot scores, and geographic diffusion data enables a new class of research: not just is this fake? but how does it spread, who spreads it, and where?

Data Architecture

Four Dimensions of Information

FakeNewsNet's defining characteristic is its multi-modal, multi-dimensional architecture. Each news item is not a row in a CSV — it is a richly annotated node in a social graph with textual content, engagement histories, visual assets, and timestamped location traces. The repository is structured as a combination of CSV files (for the minimalistic Kaggle version) and JSON directories (for the full hydrated version).

📰
Textual Content
Article title, full body text, news URL, publisher domain, publication timestamp. Primary input for NLP and transformer-based classifiers.
🌐
Social Context
Full tweet propagation tree: original posts, retweets, replies, likes. User profile attributes: bio, follower count, account age, verification status.
🤖
Bot Detection
Botometer scores for each user who shared the article. Fake news shows ~22% bot share vs. ~9% for real news — a powerful auxiliary feature.
📍
Spatiotemporal
Time-stamped tweet locations enabling diffusion chain reconstruction, cascade modelling, and geographic spread analysis.
🖼️
Visual Features
Images embedded in or linked from news articles — enabling image-text fusion models and visual misinformation research.
🔗
Network Graph
Follower/followee networks for users who shared each article — enabling GNN-based propagation and community detection experiments.
Kaggle Version vs. Full Repository

The Kaggle listing (mdepak/fakenewsnet) provides the minimalistic CSV version: four CSV files with news IDs, titles, and labels. The full multi-modal dataset — including tweet trees, user JSON files, images, and propagation graphs — must be downloaded via the KaiDMML/FakeNewsNet GitHub repository using hydration scripts and your own Twitter API credentials (required by Twitter's data distribution policy).

Critical — Tweet Hydration Required for Full Dataset

The Kaggle CSV files contain only tweet IDs, not the tweet content. To use social context features, you must run the FakeNewsNet hydration scripts against the Twitter API v2 with your own API credentials. Twitter's Academic Research access tier is required for historical tweet lookup. Tweet objects for articles from 2014–2018 may be partially unavailable due to tweet deletion and account suspension. Plan for up to 20–30% data loss during hydration.

4
Distribution Analysis

Class Imbalance Across Both Subsets

FakeNewsNet's two subsets exhibit radically different class distributions — a critical design consideration that affects every modelling decision. PolitiFact is nearly balanced (43% fake / 57% real), making it suitable for standard training regimes. GossipCop is heavily skewed (9% fake / 91% real), demanding class-weighting, oversampling (SMOTE, data augmentation), or threshold calibration before any meaningful classifier can be trained.

GossipCop's 91% Accuracy Trap

On GossipCop, a model that always predicts "real" achieves 91% accuracy. This is not a model — it's the prior distribution. Never report only accuracy on this subset. Macro-F1, fake-class recall, and AUC are mandatory metrics. The class imbalance reflects real-world fake news prevalence, making GossipCop more ecologically valid than artificially balanced datasets, but harder to benchmark.

5
Implementation Guide

Loading, Merging & Modelling

FakeNewsNet's implementation varies significantly depending on which modalities you intend to use. The CSV-only path (Kaggle minimalistic version) is straightforward — load, label, split, vectorize. The full multi-modal path requires Twitter API hydration, JSON parsing, graph construction, and optional image feature extraction. Below covers both tracks.

01
Load & Label CSVs
Load all four CSVs. Add label (0/1) and source (politifact/gossipcop) columns. Keep subsets separate for evaluation — do not merge PolitiFact and GossipCop into a single training pool without domain flags.
02
Handle Class Imbalance
For GossipCop (9% fake): apply class-weighted loss (weight ≈ 10× for fake class), or use SMOTE on text embeddings, or oversample fake rows. For PolitiFact (43%/57%): standard stratified split is sufficient without resampling.
03
Twitter Hydration (Full)
Clone KaiDMML/FakeNewsNet, configure Twitter API credentials in tweet_keys_file.json, run collection scripts. Plan for 24–72h download time and 20–30% data loss from deleted tweets. Store hydrated JSON locally — do not redistribute.
04
Feature Extraction
Text: TF-IDF, BERT embeddings, or RoBERTa. Social: aggregate tweet stats (retweet speed, bot ratio, reply sentiment). Graph: GNN on follower network. Fusion: concatenate modality embeddings before classification head.
# FakeNewsNet — CSV Loading, Labelling & Multi-Subset Preparation
import pandas as pd
import numpy as np
from sklearn.model_selection import train_test_split

# ── 1. Load all four CSVs ──────────────────────────────────────────────
poli_fake = pd.read_csv("politifact_fake.csv")
poli_real = pd.read_csv("politifact_real.csv")
goss_fake = pd.read_csv("gossipcop_fake.csv")
goss_real = pd.read_csv("gossipcop_real.csv")

# ── 2. Assign labels and source tags ──────────────────────────────────
def tag(df, label, source):
    df = df.copy()
    df["label"]  = label   # 1 = fake, 0 = real
    df["source"] = source
    return df

poli_fake = tag(poli_fake, 1, "politifact")
poli_real = tag(poli_real, 0, "politifact")
goss_fake = tag(goss_fake, 1, "gossipcop")
goss_real = tag(goss_real, 0, "gossipcop")

# ── 3. Build per-domain DataFrames ───────────────────────────────────
df_poli = pd.concat([poli_fake, poli_real], ignore_index=True).sample(frac=1, random_state=42)
df_goss = pd.concat([goss_fake, goss_real], ignore_index=True).sample(frac=1, random_state=42)

print("PolitiFact shape:", df_poli.shape)   # (783, 5)
print("GossipCop  shape:", df_goss.shape)   # (18417, 5)
print("\nPolitiFact class dist:\n", df_poli["label"].value_counts())
print("\nGossipCop class dist:\n",  df_goss["label"].value_counts())

# ── 4. Stratified splits ──────────────────────────────────────────────
def make_splits(df, test_size=0.2):
    train, temp = train_test_split(df, test_size=test_size,
                                   stratify=df["label"], random_state=42)
    val, test   = train_test_split(temp, test_size=0.5,
                                   stratify=temp["label"], random_state=42)
    return train, val, test

poli_train, poli_val, poli_test = make_splits(df_poli)
goss_train, goss_val,  goss_test  = make_splits(df_goss)

# ── 5. Class weights for GossipCop imbalance ─────────────────────────
from sklearn.utils.class_weight import compute_class_weight

weights = compute_class_weight("balanced",
    classes=np.array([0, 1]),
    y=goss_train["label"].values
)
print(f"\nGossipCop class weights: real={weights[0]:.3f}, fake={weights[1]:.3f}")
# → GossipCop class weights: real=0.549, fake=5.584
# BERT Fine-tuning on FakeNewsNet (title-only, both subsets)
from transformers import AutoTokenizer, AutoModelForSequenceClassification, Trainer, TrainingArguments
from datasets import Dataset
import torch, evaluate, numpy as np

MODEL = "roberta-base"   # RoBERTa outperforms BERT on both FNN subsets
tokenizer = AutoTokenizer.from_pretrained(MODEL)

def tokenise(batch):
    return tokenizer(batch["title"], truncation=True,
                      padding="max_length", max_length=128)

# Demonstrates on GossipCop (use same pattern for PolitiFact)
hf_train = Dataset.from_pandas(goss_train[["title","label"]].rename(columns={"label":"labels"}))
hf_val   = Dataset.from_pandas(goss_val[["title","label"]].rename(columns={"label":"labels"}))
hf_test  = Dataset.from_pandas(goss_test[["title","label"]].rename(columns={"label":"labels"}))

hf_train = hf_train.map(tokenise, batched=True)
hf_val   = hf_val.map(tokenise, batched=True)
hf_test  = hf_test.map(tokenise, batched=True)

model = AutoModelForSequenceClassification.from_pretrained(MODEL, num_labels=2)

# Weighted loss for GossipCop imbalance
class_weights_tensor = torch.tensor(weights, dtype=torch.float)

class WeightedTrainer(Trainer):
    def compute_loss(self, model, inputs, return_outputs=False):
        labels = inputs.pop("labels")
        outputs = model(**inputs)
        logits  = outputs.logits
        loss_fn = torch.nn.CrossEntropyLoss(weight=class_weights_tensor.to(model.device))
        loss = loss_fn(logits, labels)
        return (loss, outputs) if return_outputs else loss

# Evaluation: macro F1 + per-class metrics critical for imbalanced data
f1_metric = evaluate.load("f1")
acc_metric = evaluate.load("accuracy")

def compute_metrics(eval_pred):
    logits, labels = eval_pred
    preds = np.argmax(logits, axis=-1)
    return {
        "accuracy": acc_metric.compute(predictions=preds, references=labels)["accuracy"],
        "f1_macro": f1_metric.compute(predictions=preds, references=labels, average="macro")["f1"],
        "f1_fake":  f1_metric.compute(predictions=preds, references=labels, average="binary")["f1"],
    }

args = TrainingArguments(
    output_dir="./fnn-roberta-gossipcop",
    num_train_epochs=5,
    per_device_train_batch_size=16,
    per_device_eval_batch_size=32,
    learning_rate=2e-5,
    evaluation_strategy="epoch",
    save_strategy="epoch",
    load_best_model_at_end=True,
    metric_for_best_model="f1_macro",   # NOT accuracy
    warmup_ratio=0.1,
    weight_decay=0.01,
)

trainer = WeightedTrainer(
    model=model, args=args,
    train_dataset=hf_train, eval_dataset=hf_val,
    compute_metrics=compute_metrics,
)
trainer.train()
results = trainer.evaluate(hf_test)
print(f"Test Accuracy:  {results['eval_accuracy']:.4f}")
print(f"Test F1 Macro:  {results['eval_f1_macro']:.4f}")
print(f"Test F1 Fake:   {results['eval_f1_fake']:.4f}")
# Social Context Feature Extraction (after hydration)
import json, os, glob
from collections import defaultdict
import numpy as np

def extract_social_features(article_dir: str) -> dict:
    """Extract aggregate social features for one article's directory."""
    tweets_path = os.path.join(article_dir, "tweets")
    users_path  = os.path.join(article_dir, "users")
    features = defaultdict(float)

    tweet_files = glob.glob(f"{tweets_path}/*.json")
    features["tweet_count"] = len(tweet_files)

    retweet_counts, likes, reply_counts = [], [], []
    timestamps = []
    bot_scores  = []

    for tf in tweet_files:
        with open(tf) as f:
            t = json.load(f)
        retweet_counts.append(t.get("retweet_count", 0))
        likes.append(t.get("favorite_count", 0))
        reply_counts.append(t.get("reply_count", 0))
        timestamps.append(t.get("created_at", ""))

    # Spread velocity: max retweets in first hour (proxy)
    features["avg_retweets"]  = np.mean(retweet_counts) if retweet_counts else 0
    features["avg_likes"]     = np.mean(likes) if likes else 0
    features["total_replies"]  = sum(reply_counts)

    # User bot scores
    user_files = glob.glob(f"{users_path}/*.json")
    for uf in user_files:
        with open(uf) as f:
            u = json.load(f)
        if "bot_score" in u:
            bot_scores.append(u["bot_score"])

    features["bot_ratio"] = np.mean([1 if s > 0.5 else 0 for s in bot_scores]) if bot_scores else 0
    features["mean_bot_score"] = np.mean(bot_scores) if bot_scores else 0

    return dict(features)

The E! Online Leakage — GossipCop's Silent Confounder

All 16,767 real GossipCop articles originate from E! Online. Models can learn URL domain, writing style, or even image size/aspect ratios associated with E! Online content. This inflates GossipCop accuracy in the same way Reuters bylines inflate ISOT results. Always strip or hash the news_url field before training, and validate that model attention is not disproportionately focused on URL tokens.

8
Limitations & Ethical Considerations

What FakeNewsNet Cannot Tell You

🐦
Twitter API Dependency — Chronic Data Loss
The full multi-modal dataset requires Twitter API hydration. Since 2014–2018 data collection, an estimated 20–30% of original tweets have been deleted or made private, and accounts have been suspended. Elon Musk's 2023 API restructuring further restricted Academic Research access. Any experiment using social features will operate on an incomplete cascade graph — results are not reproducible without documenting hydration date and data loss rate.
⚖️
Binarization of Nuanced Verdicts
PolitiFact uses a 6-point scale: Pants on Fire, False, Mostly False, Half True, Mostly True, True. FakeNewsNet collapses this to binary: False/Mostly False/Pants-on-Fire → fake; True → real. "Half True" and "Mostly True" articles are excluded entirely. This simplification loses the epistemic gradation that makes misinformation research meaningful — and it means label=0 (real) is a very high-confidence ground truth, while label=1 (fake) conflates egregious lies with nuanced misrepresentations.
🌐
Domain Homogeneity Within Each Subset
PolitiFact covers exclusively US domestic political claims. GossipCop covers exclusively US celebrity entertainment. Neither subset includes science misinformation, health claims, financial fraud, international disinformation, or social media posts. Cross-domain generalization from this dataset to any other information domain is poor — empirically measured at 55–62% accuracy for PolitiFact→GossipCop transfers and vice versa.
🌐
English-Only, US-Centric Corpus
All articles are in English, published in US media contexts. Fact-checking standards, cultural references, rhetorical norms, and bot behavior differ substantially across languages and regions. Models trained on FakeNewsNet should not be applied to non-English or non-US contexts without extensive domain adaptation and revalidation.
⏰
Temporal Obsolescence
Coverage spans 2014–2018. The misinformation ecosystem of 2024–2025 — dominated by LLM-generated synthetic text, deepfake multimedia, cross-platform coordinated inauthentic behavior, and influence operations at industrial scale — is entirely absent. Bot detection models trained on 2016 Botometer scores will not generalize to current AI-assisted bot behavior patterns.
🤖
Social Context is Correlational, Not Causal
Bot ratio, retweet velocity, and engagement patterns are correlates of fake news — not causes. A model that classifies primarily on social signals will fail on breaking news (high bot activity regardless of veracity), viral satire, and astroturfed real news. Social features should be fusion inputs, not primary classifiers, and their temporal availability must be modelled (you don't have retweet data at publication time).
Responsible Use Guidance

FakeNewsNet is excellent for multi-modal fake news research prototyping, propagation analysis, and social network modelling. It is not suitable as the sole training data for production content moderation systems — the temporal staleness, Twitter dependency, and domain narrowness make it an unsafe foundation for consequential automated decisions. Always combine with up-to-date, domain-matched data and human annotation layers.

Which Dataset Should You Use?

Choose FakeNewsNet for multi-modal research, propagation modelling, or cross-domain experiments. Choose ISOT FND for teaching NLP fundamentals and testing text preprocessing pipelines (with explicit leakage disclaimers). Choose LIAR for serious veracity grading research that resists shortcut learning. For production systems: combine all three with domain-specific contemporary data and human annotation layers.
