01
Introduction

What is the ISOT Dataset?

The ISOT Fake News Dataset is a benchmark corpus for binary news veracity classification, widely adopted in NLP research since its release. Curated by the Information Security and Object Technology (ISOT) Research Lab at the University of Victoria, Canada, it provides a strictly balanced collection of real world political news. Access the full dataset on Kaggle .

Real articles were scraped directly from Reuters.com, one of the world's most respected wire services with strict editorial standards, professional bylines, and fact-checking workflows. Fake articles were collected from sites explicitly flagged as unreliable by organisations including Politifact, Wikipedia's list of fake news websites, and established fact-checking outlets. Every fake article was subsequently evaluated by trained journalists before inclusion.

The dataset ships as two CSV files — True.csv and Fake.csv — each with four fields: title, text, subject, and date. The label is implicit in which file a record comes from.

The Most Important Warning in This Report

The subject field is a near-perfect label leaker. Fake articles are overwhelmingly tagged "News" while real articles span "politicsNews" and "worldnews". Any model trained with the subject field included achieves >99% accuracy — but it is measuring topic classification, not veracity detection. Always drop the subject field and cross-validate across unseen publishers before claiming results.

Research Insight — The Reuters Dateline Problem

Because every real article begins with "CITY (Reuters) -", a model that simply checks for the presence of "(Reuters)" in the first 30 characters achieves near-perfect accuracy with zero machine learning. This makes ISOT deceptively easy. For meaningful results, strip the dateline prefix, drop the subject field, and test on cross-publisher splits where the model cannot rely on publication style cues.

Temporal Coverage

Article Volume Over Time

The dataset spans January 2016 to December 2017 — two full years covering the 2016 US presidential election, the Trump transition, and the first year of the Trump presidency. Article volume grows dramatically over time, with the densest coverage in Q3–Q4 2017 as political news activity intensified.

Approximate Article Volume by Period (True.csv — Reuters Coverage)
Temporal Split Recommendation

For a realistic evaluation, split chronologically: train on 2016 and early 2017, test on late 2017. This simulates deployment conditions far better than random splits and tests whether detectors generalise across political cycles, not just across randomly held-out articles from the same time period.

05
Implementation

Loading, Cleaning & Modelling

01
Load Both CSVs
Read True.csv and Fake.csv separately with pandas. Assign labels (0/1) before concatenating. Shuffle the combined DataFrame to break file-order bias.
02
Drop Subject Field
Remove the subject column entirely. If you must use it, explicitly ablate and document results both with and without — subject-included results are not comparable to cross-dataset benchmarks.
03
Strip Dateline
Remove the "CITY (Reuters) -" prefix from real article text using regex. This prevents the model from exploiting the publication-style signal rather than learning semantic content.
04
Temporal Split
Parse the date column and sort chronologically. Use 70/15/15 train/val/test with the test set covering the most recent dates — not random splits.
# ISOT Dataset — Complete Load, Clean & Feature Engineering Pipeline
import pandas as pd
import re, numpy as np
from sklearn.model_selection import train_test_split

# 1. Load both files and assign labels
real = pd.read_csv("./True.csv").assign(label=0, source="reuters")
fake = pd.read_csv("./Fake.csv").assign(label=1, source="unreliable")

df = pd.concat([real, fake], ignore_index=True)
df = df.sample(frac=1, random_state=42).reset_index(drop=True)

# 2. Drop subject (label leaker!) — CRITICAL
df = df.drop(columns=["subject"])

# 3. Strip Reuters dateline from real article text
def strip_dateline(text: str) -> str:
    # matches "CITY (Reuters) -" and "CITY/CITY (Reuters) -"
    pattern = r'^[A-Z\s/]+ \(Reuters\)\s*[-–]\s*'
    return re.sub(pattern, '', str(text)).strip()

df["text_clean"] = df["text"].apply(strip_dateline)

# 4. Build content field (title + cleaned text)
df["content"] = df["title"].fillna("") + " [SEP] " + df["text_clean"].fillna("")

# 5. Parse date for temporal split
df["date_parsed"] = pd.to_datetime(df["date"], format="%B %d, %Y", errors="coerce")
df = df.sort_values("date_parsed").reset_index(drop=True)

# 6. Chronological train / val / test split
n = len(df)
train_df = df.iloc[:int(n * 0.70)]
val_df   = df.iloc[int(n * 0.70):int(n * 0.85)]
test_df  = df.iloc[int(n * 0.85):]

print(f"Train: {len(train_df)} | Val: {len(val_df)} | Test: {len(test_df)}")
print(f"Train date range: {train_df['date_parsed'].min()} → {train_df['date_parsed'].max()}")
print(f"Test  date range: {test_df['date_parsed'].min()} → {test_df['date_parsed'].max()}")
print(train_df["label"].value_counts())
# BERT fine-tuning on cleaned ISOT text
from transformers import AutoTokenizer, AutoModelForSequenceClassification, Trainer, TrainingArguments
from datasets import Dataset

MODEL = "bert-base-uncased"
tokenizer = AutoTokenizer.from_pretrained(MODEL)

def tok(batch):
    return tokenizer(batch["content"],
                     truncation=True, padding="max_length",
                     max_length=512)

def make_ds(df):
    ds = Dataset.from_pandas(df[["content","label"]])
    return ds.map(tok, batched=True)

hf_train = make_ds(train_df)
hf_val   = make_ds(val_df)
hf_test  = make_ds(test_df)

model = AutoModelForSequenceClassification.from_pretrained(MODEL, num_labels=2)

args = TrainingArguments(
    output_dir="./isot-bert",
    num_train_epochs=3,
    per_device_train_batch_size=16,
    evaluation_strategy="epoch",
    learning_rate=2e-5,
    weight_decay=0.01,
    load_best_model_at_end=True,
    warmup_ratio=0.06,
    save_strategy="epoch",
)

trainer = Trainer(model=model, args=args,
                  train_dataset=hf_train, eval_dataset=hf_val)
trainer.train()
results = trainer.evaluate(hf_test)
print(results)

Published Results

Model Benchmark Landscape

ISOT's near-100% BERT accuracy is the most widely cited — and most widely misinterpreted — result in fake news research. The figures below reflect results with the subject field dropped and a random 80/20 train/test split, as used in most published papers. Chronological split accuracy is typically 5–10 points lower.

Model Features Split Type Accuracy F1 (Fake) Notes
Naive Bayes (TF-IDF) Unigrams Random 80/20 92–94% ~0.93 Simple but strong baseline
Logistic Regression TF-IDF + bigrams Random 80/20 94–96% ~0.95 Fast, interpretable, competitive
BiLSTM (GloVe) Word embeddings Random 80/20 97–98% ~0.97 Seq. model captures headline patterns
BERT (no dateline strip) title + text Random 80/20 >99% ~0.99 Learns "Reuters" token — misleading
BERT (dateline stripped) title + text_clean Random 80/20 97–98% ~0.97 More honest result
BERT (dateline stripped) title + text_clean Chronological 90–94% ~0.91 Most realistic deployment estimate
Cross-dataset (→FakeNewsNet) BERT fine-tuned on ISOT Transfer 60–75% ~0.65 Reveals domain-specific overfitting
Benchmark Credibility Warning

Papers reporting >99% accuracy on ISOT without specifying (a) whether subject was dropped, (b) whether dateline was stripped, and (c) what split type was used should be treated with scepticism. These three variables alone account for a ~10 percentage-point spread in results. Always report all three conditions explicitly.

07
Critical Analysis

Limitations & Responsible Use

!
Source-Level Labels
Labels are assigned at the outlet level — all Reuters = real, all flagged sites = fake. This means a model learns to detect publication style, not claim veracity. An article from a flagged site that happens to be factually correct is still labelled fake.
!
US Politics Concentration
Both files focus heavily on 2016–2017 US political news. Coverage of international, scientific, health, or economic misinformation is thin. Models trained here transfer poorly to non-political domains.
!
Static Snapshot
The dataset ends December 2017. Post-2017 generative AI content, new misinformation patterns, and evolved fact-checking norms are entirely absent. Not suitable as a sole training set for production systems.
!
No Fine-Grained Labels
Binary only — no satire, no opinion, no "misleading but not false" class. An article that is technically true but deliberately framed to deceive is labelled the same as one that is factually wrong.
